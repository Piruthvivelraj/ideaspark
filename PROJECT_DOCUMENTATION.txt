1. WHAT WAS BUILT
================================================================================

IdeaSpark is a full-stack web application that guides students through a
7-question conversational wizard and then recommends matching project ideas
from a curated dataset of 100 projects across 5 domains.

Key capabilities:
  - User registration and login with secure authentication
  - Step-by-step wizard that collects user preferences one question at a time
  - Smart idea matching based on domain and difficulty
  - No-repeat system: ideas already seen by a user are never shown again
  - Animated idea cards with typewriter description effect
  - Detailed project page with flow, features, tech stack, and difficulty meter
  - Personal dashboard with stats, domain breakdown, and recent saved ideas
  - Saved ideas page where users can bookmark and delete ideas
  - Fully responsive, student-friendly UI with vivid purple/pink color palette

2. TECHNOLOGIES USED (WITH DEFINITIONS)
================================================================================

--- FRONTEND ---

React.js
  A JavaScript library for building user interfaces using reusable components.
  It updates only the parts of the page that change, making apps fast and smooth.
  Used for: All pages and UI components in this project.

React Router DOM
  A library that enables navigation between pages in a React app without
  reloading the browser. It maps URLs to components.
  Used for: Routing between Home, Login, Register, Dashboard, Saved, Project Detail.

Tailwind CSS
  A utility-first CSS framework where you style elements directly in HTML/JSX
  using class names like "text-white", "flex", "rounded-xl" instead of writing
  separate CSS files.
  Used for: All styling, layout, spacing, and responsive design.

Framer Motion
  A React animation library that makes it easy to add smooth animations,
  transitions, and gesture effects to components.
  Used for: Animated background blobs, card entrance animations, typewriter
  effect, step transitions in the wizard, hover effects.

Axios
  A JavaScript library for making HTTP requests (GET, POST, DELETE) from the
  browser to a backend server. Simpler than the built-in fetch API.
  Used for: All API calls from the frontend to the Express backend.

React Hot Toast
  A lightweight notification library for React that shows small popup messages
  (toasts) for success, error, or info events.
  Used for: "Idea saved!", "Already saved", "Failed to generate" messages.

Lucide React
  A library of clean, consistent SVG icons for React applications.
  Used for: Icons like Bookmark, ArrowLeft, Copy, Zap, Globe, BarChart2, etc.

--- BACKEND ---

Node.js
  A JavaScript runtime that allows JavaScript to run on the server (outside
  the browser). It is fast, lightweight, and event-driven.
  Used for: Running the backend server.

Express.js
  A minimal web framework for Node.js that makes it easy to define API routes,
  handle requests, and send responses.
  Used for: All backend API routes (/auth/register, /auth/login, /ideas/generate,
  /ideas/save, /ideas/saved, /ideas/delete/:id).

bcryptjs
  A library for hashing (encrypting) passwords before storing them in the
  database. Hashing is one-way — the original password cannot be recovered.
  Used for: Hashing user passwords during registration and comparing during login.

JSON Web Token (JWT)
  A compact, secure token format used for authentication. After login, the
  server issues a JWT. The client sends this token with every request to prove
  identity without logging in again.
  Used for: Protecting routes so only logged-in users can generate or save ideas.

dotenv
  A Node.js package that loads environment variables from a .env file into
  process.env. Keeps sensitive data like secrets and DB URLs out of source code.
  Used for: Storing MONGO_URI, JWT_SECRET, PORT.

CORS (cors package)
  Cross-Origin Resource Sharing. A security mechanism that controls which
  domains can access your backend API. The cors package configures this easily.
  Used for: Allowing the React frontend (localhost:3000) to call the Express
  backend (localhost:5000).

--- DATABASE ---

MongoDB
  A NoSQL database that stores data as flexible JSON-like documents instead of
  rigid tables and rows. Great for rapidly changing data structures.
  Used for: Storing users, saved ideas, and seen ideas.

Mongoose
  An ODM (Object Data Modeling) library for MongoDB and Node.js. It lets you
  define schemas (structure/rules) for your data and interact with MongoDB
  using JavaScript objects.
  Used for: Defining User, Idea, and SeenIdea models and querying the database.

4. WORKING FLOW (STEP BY STEP)
================================================================================

STEP 1 — Register
  User visits /register and fills in Name, Email, Password.
  Password is hashed using bcryptjs and stored in MongoDB.
  A JWT token is returned and saved in localStorage.

STEP 2 — Login
  User visits /login and enters Email, Password.
  Server compares the entered password with the stored hash using bcrypt.compare().
  If valid, a JWT token is issued and stored in localStorage.

STEP 3 — Answer the 7-Question Wizard (Home Page)
  The wizard asks one question at a time with animated transitions:
    Q1. Which domain are you interested in?
        (AI / Web Development / IoT / Mobile App / Cybersecurity)
    Q2. What is your experience level?
        (Beginner / Intermediate / Advanced / Any)
    Q3. What is your main goal?
        (Learn Skills / Build Portfolio / Win Competition / Solve a Problem)
    Q4. Are you working solo or in a team?
        (Solo / Small Team / Large Team)
    Q5. What is your project timeline?
        (1–3 Days / 1–2 Weeks / 1 Month / 3+ Months)
    Q6. Which technology are you most comfortable with?
        (Python / JavaScript / Java / C++ / No Preference)
    Q7. What type of project output do you prefer?
        (Web App / Mobile App / Hardware / ML Model / Tool/Script)

  A progress bar at the top shows how many questions are answered.
  Previously selected answers are shown as tags above the current question.
  Users can go back to change a previous answer.

STEP 4 — Idea Generation
  After the last question, the frontend sends a GET request to:
    /api/ideas/generate?domain=<domain>&difficulty=<difficulty>
  The server filters the predefined dataset by domain and difficulty.
  Ideas the user has already seen (tracked in SeenIdea collection) are excluded.
  Newly shown ideas are recorded in SeenIdea so they won't repeat next time.
  Matching ideas are returned and displayed as animated cards.

STEP 5 — View Idea Cards
  Each card shows: title, domain tag, difficulty badge, tech stack chips.
  Cards animate in with staggered entrance (each card appears slightly after the previous).
  The description types out character by character (typewriter effect).
  Hovering a card shows a colored glow based on difficulty level.

STEP 6 — View Project Detail
  Clicking a card navigates to /project with the idea data passed via router state.
  The detail page shows:
    - Project title, domain, difficulty badge
    - Quick stats: difficulty, domain, estimated timeline, team size
    - Project Flow: 6 universal steps (Setup → Design → Development → Frontend
      → Testing → Deployment) shown as a vertical timeline
    - Key Features: 5 domain-specific features listed
    - Tech Stack: all technologies listed as interactive chips
    - Difficulty Meter: visual bar showing Beginner / Intermediate / Advanced
    - Save Idea button and Copy to Clipboard button

STEP 7 — Save an Idea
  Clicking "Save Idea" sends a POST to /api/ideas/save with the idea data.
  The idea is stored in the Idea collection in MongoDB linked to the user's ID.
  If already saved, the server returns "Already saved" and a toast is shown.

STEP 8 — Dashboard
  Accessible via /dashboard from the Navbar.
  Shows animated stat counters: total saved, domains explored, beginner count,
  advanced count.
  Lists the 5 most recently saved ideas.
  Shows a domain breakdown bar chart of saved ideas.
  Quick action buttons to generate more ideas or view all saved ideas.

STEP 9 — Saved Ideas Page
  Accessible via /saved from the Navbar.
  Shows all saved ideas as cards with domain, difficulty, and tech stack.
  Each card has a Delete button that sends DELETE /api/ideas/:id.
  The idea is removed from MongoDB and the UI updates instantly.

STEP 10 — Logout
  Clicking Logout in the Navbar clears the JWT from localStorage.
  The user is redirected to the Login page.
  All protected routes redirect to /login if no valid token is present.

5. DATA STORED IN MONGODB (DATABASE COLLECTIONS)
================================================================================

DATABASE NAME: (default, set in MONGO_URI in .env)
CONNECTION: MongoDB running on localhost:27017

--- COLLECTION 1: users ---
Stores registered user accounts.

Fields:
  _id          ObjectId    Auto-generated unique ID by MongoDB
  name         String      Full name of the user (required)
  email        String      Email address, must be unique (required)
  password     String      Bcrypt-hashed password (required)
  createdAt    Date        Auto-added by Mongoose timestamps
  updatedAt    Date        Auto-added by Mongoose timestamps

Example document:
  {
    "_id": ObjectId("..."),
    "name": "Arun Kumar",
    "email": "arun@example.com",
    "password": "$2b$10$hashedpasswordhere...",
    "createdAt": ISODate("2025-01-01T10:00:00Z")
  }

--- COLLECTION 2: ideas ---
Stores project ideas that users have bookmarked/saved.

Fields:
  _id          ObjectId    Auto-generated unique ID
  userId       ObjectId    Reference to the User who saved this idea
  title        String      Project title
  description  String      Short description of the project
  techStack    [String]    Array of technology names
  difficulty   String      "Beginner", "Intermediate", or "Advanced"
  domain       String      Domain name (e.g., "Artificial Intelligence")
  savedAt      Date        Timestamp when the idea was saved (default: now)

Example document:
  {
    "_id": ObjectId("..."),
    "userId": ObjectId("..."),
    "title": "Smart Attendance System using Face Recognition",
    "description": "Automate attendance tracking using real-time face detection.",
    "techStack": ["Python", "OpenCV", "TensorFlow", "Flask", "MongoDB"],
    "difficulty": "Intermediate",
    "domain": "Artificial Intelligence",
    "savedAt": ISODate("2025-01-02T12:00:00Z")
  }

--- COLLECTION 3: seenideas ---
Tracks which ideas a user has already been shown, to prevent repetition.

Fields:
  _id          ObjectId    Auto-generated unique ID
  userId       ObjectId    Reference to the User
  domain       String      Domain of the idea that was shown
  title        String      Title of the idea that was shown
  seenAt       Date        When the idea was shown (default: now)

Unique Index: { userId, domain, title } — ensures no duplicate seen records.

Example document:
  {
    "_id": ObjectId("..."),
    "userId": ObjectId("..."),
    "domain": "Web Development",
    "title": "Real-Time Chat Application",
    "seenAt": ISODate("2025-01-03T09:30:00Z")
  }

7. API ROUTES SUMMARY
================================================================================

AUTH ROUTES (no token required):
  POST  /api/auth/register    Register a new user
  POST  /api/auth/login       Login and receive a JWT token

IDEA ROUTES (JWT token required in Authorization header):
  GET   /api/ideas/generate   Get matching ideas (filters by domain + difficulty,
                              excludes already-seen ideas, records new seen ideas)
  POST  /api/ideas/save       Save a project idea to the user's collection
  GET   /api/ideas/saved      Get all saved ideas for the logged-in user
  DELETE /api/ideas/:id       Delete a saved idea by its MongoDB ID

8. HOW TO RUN THE PROJECT
================================================================================

PREREQUISITES:
  - Node.js installed
  - MongoDB running on localhost:27017

START BACKEND:
  cd ai-project-generator/server
  npm install
  npm start
  (Runs on http://localhost:5000)

START FRONTEND:
  cd ai-project-generator/client
  npm install
  npm start
  (Runs on http://localhost:3000)

ENVIRONMENT VARIABLES (server/.env):
  MONGO_URI=mongodb://localhost:27017/ideaspark
  JWT_SECRET=your_secret_key_here
  PORT=5000

9. PROJECT SUMMARY
================================================================================

IdeaSpark was built as a hackathon project to solve the problem of students
struggling to find suitable project ideas. Instead of using an AI API (which
requires paid keys), the system uses a carefully curated dataset of 100 real,
practical project ideas across 5 popular domains.

The conversational wizard makes the experience feel personalized and engaging.
The no-repeat system ensures students always discover new ideas across sessions.
The dashboard and saved ideas features help students track and revisit their
shortlisted projects.

The tech stack (React + Tailwind + Framer Motion on the frontend, Node.js +
Express + MongoDB on the backend) is a modern, industry-standard combination
that is also beginner-friendly and well-documented.
